# PWA (Progressive Web App) Setup Guide

## Overview

The Sierra Suites is configured as a Progressive Web App (PWA), allowing users to install it on their devices for a native app-like experience.

## Features Enabled

### ✅ Offline Support
- Service worker caches critical assets
- Previously viewed pages available offline
- Offline fallback page at `/offline`
- Graceful degradation when network is unavailable

### ✅ Installability
- Can be installed on iOS, Android, and desktop
- Appears in app drawer/home screen
- Standalone window (no browser chrome)
- Splash screen on launch

### ✅ Caching Strategy

**Images** (`CacheFirst`, 30 days)
- PNG, JPG, JPEG, SVG, GIF, WEBP, AVIF

**Fonts** (`CacheFirst`, 1 year)
- WOFF, WOFF2, TTF, OTF, EOT
- Google Fonts

**API Calls** (`NetworkFirst`, 5 minutes)
- `/api/*` endpoints
- Supabase API calls (1 hour cache)

### ✅ App Shortcuts
Quick access to common actions:
- New Project
- New Quote
- Capture Photo (FieldSnap)

## Installation

### On Mobile (iOS/Android)

**iOS (Safari):**
1. Open https://yourdomain.com in Safari
2. Tap the Share button
3. Scroll down and tap "Add to Home Screen"
4. Tap "Add"

**Android (Chrome):**
1. Open https://yourdomain.com in Chrome
2. Tap the menu (⋮)
3. Tap "Install app" or "Add to Home Screen"
4. Tap "Install"

### On Desktop (Chrome/Edge)

1. Open https://yourdomain.com
2. Look for the install icon (+) in the address bar
3. Click it and confirm installation
4. App appears in your applications folder

## App Icons

The following icon sizes are required and should be placed in `/public/icons/`:

- `icon-72x72.png`
- `icon-96x96.png`
- `icon-128x128.png`
- `icon-144x144.png`
- `icon-152x152.png`
- `icon-192x192.png`
- `icon-384x384.png`
- `icon-512x512.png`

**Icon Requirements:**
- Square aspect ratio
- Transparent or solid background
- Simple, recognizable design
- High contrast for visibility

### Generating Icons

You can generate all required sizes from a single SVG or large PNG:

```bash
# Using ImageMagick (if installed)
convert icon-original.png -resize 72x72 public/icons/icon-72x72.png
convert icon-original.png -resize 96x96 public/icons/icon-96x96.png
convert icon-original.png -resize 128x128 public/icons/icon-128x128.png
convert icon-original.png -resize 144x144 public/icons/icon-144x144.png
convert icon-original.png -resize 152x152 public/icons/icon-152x152.png
convert icon-original.png -resize 192x192 public/icons/icon-192x192.png
convert icon-original.png -resize 384x384 public/icons/icon-384x384.png
convert icon-original.png -resize 512x512 public/icons/icon-512x512.png
```

Or use an online tool like [PWA Asset Generator](https://www.pwabuilder.com/imageGenerator).

## Configuration

### Manifest File

Located at `/public/manifest.json`

Key settings:
- **name**: Full app name
- **short_name**: Name shown on home screen
- **start_url**: Where app opens (`/dashboard`)
- **display**: `standalone` (no browser UI)
- **theme_color**: `#1E40AF` (blue)
- **background_color**: `#ffffff` (white)

### Service Worker

Automatically generated by `next-pwa` in `/public/sw.js`

Configuration in `next.config.mjs`:
- Disabled in development
- Enabled in production build
- Auto-registers on page load
- Skips waiting for activation

### Caching Configuration

Edit `next.config.mjs` to customize caching strategies:

```javascript
runtimeCaching: [
  {
    urlPattern: /\/api\/.*/,
    handler: 'NetworkFirst', // Try network, fallback to cache
    options: {
      cacheName: 'api-cache',
      expiration: {
        maxEntries: 50,
        maxAgeSeconds: 5 * 60, // 5 minutes
      },
    },
  },
]
```

**Available Handlers:**
- `NetworkFirst` - Network with cache fallback (good for API calls)
- `CacheFirst` - Cache with network fallback (good for static assets)
- `NetworkOnly` - Always fetch from network (good for critical data)
- `CacheOnly` - Always use cache (good for offline-first)
- `StaleWhileRevalidate` - Use cache immediately, update in background

## Troubleshooting

### Service Worker Not Registering

- Check browser console for errors
- Verify manifest.json is valid (use [Manifest Validator](https://manifest-validator.appspot.com/))
- Ensure app is served over HTTPS (required for PWA)
- Clear browser cache and hard reload

### Install Prompt Not Showing

- Verify all PWA requirements are met
- Check if service worker is active
- Try in incognito/private browsing mode
- Some browsers require user engagement before showing prompt

### Icons Not Loading

- Verify icons exist in `/public/icons/`
- Check file names match manifest.json exactly
- Ensure icons are valid PNG files
- Check browser console for 404 errors

### Offline Mode Not Working

- Verify service worker is active
- Check caching strategy in next.config.mjs
- Ensure URLs match caching patterns
- Check service worker cache in DevTools

5. Send notifications from backend

## App Updates

When you deploy a new version:

1. Service worker automatically detects new version
2. Downloads new assets in background
3. Activates on next app reload
4. User sees "Update available" notification (if implemented)

To force immediate update:
```javascript
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.getRegistrations().then(registrations => {
    registrations.forEach(registration => registration.update())
  })
}
```
